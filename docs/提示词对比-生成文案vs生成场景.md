# 提示词对比 - 生成文案 vs 生成场景

**日期**: 2026-01-28

## 1. 生成文案（Synopsis）

### 提示词结构

**System Prompt**:
```
你是一个专业的短视频剧本创作专家。
请根据项目信息和角色设定，生成一个简洁的剧本大概。

要求：
- 剧本大概应该是 100-300 字的简短描述
- 描述剧本的核心故事线和主要情节
- 体现角色特点和关系
- 符合短视频的节奏和风格
- 基调风格：${tone}（可选）
- 只输出剧本大概内容，不要其他说明
```

**User Prompt**:
```
项目主题：${project.topic}
主题名称：${project.themeName}（可选）
主题描述：${project.themeDesc}（可选）

角色设定：
- ${character.name}: ${character.description}

请生成剧本大概：
```

**输出格式**: 纯文本（100-300字）

**评价**: ✅ 简洁明确，适合文本生成

## 2. 生成场景（Scenes）- 优化后

### 提示词结构

**System Prompt**:
```
你是一个专业的短视频剧本创作专家。
请根据剧本大概，生成${sceneCount}个场景。

要求：
- 返回一个 JSON 数组，包含${sceneCount}个场景对象
- 每个场景对象包含：
  - title（场景标题，5-15字）
  - duration（预计时长，5-60秒）
  - content（场景内容对象）

- content 对象必须包含完整结构：
  {
    "description": "场景描述（20-50字）",
    "sceneType": "indoor/outdoor/special",
    "characters": [{
      "characterId": "角色ID",
      "characterName": "角色名称",
      "action": "动作描述",
      "emotion": "情绪状态",
      "position": "left/center/right"
    }],
    "dialogues": [{
      "characterId": "角色ID",
      "text": "台词内容",
      "speed": "slow/normal/fast",
      "tone": "语气"
    }],
    "camera": {
      "type": "closeup/medium/full/wide",
      "movement": "static/push/pull/follow"
    },
    "visual": {
      "transition": "转场效果",
      "effects": ["视觉效果"],
      "subtitleStyle": "字幕样式"
    },
    "audio": {
      "bgm": "背景音乐类型",
      "soundEffects": ["音效"]
    }
  }

场景创作要点：
- 场景要连贯，符合剧本大概的故事线
- 台词要口语化，适合短视频，每句控制在 15-30 字
- 根据场景内容合理选择镜头类型和运镜方式
- 为每个场景选择合适的背景音乐和音效
- 场景类型根据描述判断：室内用 indoor，户外用 outdoor，特殊场景用 special
- 基调风格：${tone}（可选）
- 只输出 JSON 数组，不要其他说明

[示例输出格式...]
```

**User Prompt**:
```
剧本大概：
${synopsis}

角色列表：
- ${character.name} (ID: ${character.id}): ${character.description}

请生成${sceneCount}个场景：
```

**输出格式**: JSON 数组（结构化数据）

**评价**: ✅ 完整详细，AI 可以生成高质量的结构化数据

---

## 提示词设计原则

### 1. 明确输出格式
- 文本生成 → 纯文本
- 结构化数据 → JSON + 完整示例

### 2. 提供详细约束
- 字数限制（如：100-300字，15-30字）
- 时长限制（如：5-60秒）
- 枚举值（如：indoor/outdoor/special）

### 3. 给出创作指导
- 风格要求（如：口语化、适合短视频）
- 内容要求（如：连贯、符合故事线）
- 智能选择（如：根据场景内容选择镜头）

### 4. 包含完整示例
- 展示期望的输出格式
- 包含所有必需字段
- 使用真实的示例值

## 提示词模板

### 文本生成模板
```
你是一个专业的${领域}专家。
请根据${输入信息}，生成${输出内容}。

要求：
- ${约束1}
- ${约束2}
- ${风格要求}
- 只输出${输出内容}，不要其他说明
```

### 结构化数据生成模板
```
你是一个专业的${领域}专家。
请根据${输入信息}，生成${数量}个${对象}。

要求：
- 返回一个 JSON 数组，包含${数量}个${对象}对象
- 每个对象必须包含以下完整结构：
  {
    "field1": "说明（约束）",
    "field2": "说明（枚举值）",
    ...
  }

创作要点：
- ${要点1}
- ${要点2}
- ${智能选择指导}
- 只输出 JSON 数组，不要其他说明

示例输出格式：
[完整的 JSON 示例]
```

## 总结

| 特性 | 生成文案 | 生成场景 |
|------|---------|---------|
| **输出格式** | 纯文本 | JSON 数组 |
| **复杂度** | 简单 | 复杂 |
| **字段数量** | 1 个 | 8+ 个嵌套字段 |
| **提示词长度** | 短（~100 字） | 长（~500 字） |
| **示例** | 不需要 | 必需 |
| **验证** | 字数检查 | 结构验证 + 降级 |

**关键差异**：
- 文本生成 → 简洁提示词 + 约束条件
- 结构化生成 → 详细提示词 + 完整示例 + 验证机制
